# Phase 8 Batch 40 — Cognito Setup Checklist

Use this checklist when manually preparing Cognito for RSA CMS admin authentication.

## 1. Before opening Cognito

Confirm:

- AWS region is `ap-southeast-1`.
- EC2 public site is already working through Nginx.
- Admin routes are still blocked publicly.
- Billing alert/budget is active.
- EC2 can be stopped while doing planning work.

## 2. User pool naming

Recommended names:

```text
User pool name: rsa-cms-admin-user-pool
App client name: rsa-cms-admin-web-client
Initial admin user email: use the owner/admin email only
```

## 3. Sign-in settings

Use:

```text
Sign-in identifier: Email
Required attributes: Email only
Phone number: not required
Username aliases: avoid unless needed
```

Do not use phone-number login for this phase.

## 4. Security settings

Use:

```text
MFA: Off for launch/demo unless explicitly approved
SMS MFA: Off
Phone verification: Off
Self-service public sign-up: Off or restricted
Admin-created user: preferred
```

Do not enable SMS flows for this project phase.

## 5. App client settings

Create one app client for the admin frontend.

Use a public/static frontend-compatible app client:

```text
App client name: rsa-cms-admin-web-client
Client secret: No client secret
Allowed auth flow: username/email + password style login supported by the existing admin-auth frontend
```

Record:

```text
User pool ID
App client ID
Region
```

Do not commit these values into Git if they are placed into a real EC2/runtime env file.

## 6. Admin user

Create one initial admin user manually:

```text
Email: owner/admin email
Temporary password: generated by Cognito or manually created securely
User status: force password change is acceptable for first login
```

Do not create public registration for customers in this phase.

## 7. EC2 runtime env values for Batch 41

Batch 40 keeps admin auth disabled. Batch 41 will use values like:

```text
RSA_ADMIN_AUTH_MODE=cognito
RSA_COGNITO_REGION=ap-southeast-1
RSA_COGNITO_USER_POOL_ID=<user-pool-id>
RSA_COGNITO_APP_CLIENT_ID=<app-client-id>
```

## 8. Nginx safety before Batch 41

Before enabling admin exposure, verify these remain blocked:

```text
/admin/
/api/admin/products
/api/customers
/api/bookings
/api/inquiries
/docs
/redoc
/openapi.json
```

## 9. Do not create these in Batch 40

```text
Route 53/domain
Elastic IP
ALB
NAT Gateway
RDS
SNS/SMS notification workflows
Cognito SMS MFA
Phone-number login requirement
```
